# 25. SCM-versioned CLI

Date: 2026-04-08

## Status

Accepted

## Context

The CLI version was previously stored in a manually maintained `_build_info.py`
file, which could easily fall out of sync with the actual release tag.
Consumers of generated files (e.g., OAS stubs, datapackages) benefit from
knowing which version of the tool produced them.

## Decision

- [x] Use `setuptools-scm` to derive the package version from Git tags
  at build time.
- [x] Embed the CLI version as a trailing comment in all YAML files
  generated by the CLI (via `yaml_dump`).

## Consequences

Pros:

- Version is always in sync with the Git tag; no manual step required.
- Generated files are traceable to the CLI version that produced them.

Cons:

- Building from a shallow clone or without tags yields a `0+dev` fallback version.
